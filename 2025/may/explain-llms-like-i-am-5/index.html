Explain LLMs like I am 5

  • Posted 10 May 2025

Watching an LLM spew out what seems like intellgent human level "thinking" always makes me pounder: how is this actually working?

Word Is a Pet? Is Furry? Is a sound?
Dog 0.8 0.7 0.1
Cat 0.7 0.6 0.1
Pig 0.4 0.1 0.1
Word Is a Pet? Is Cat Related? Is a sound?
Meow 0.1 0.9 0.9
Oink 0.1 0.1 0.9 

Question 1: Do dogs meow?

“dog” has a high “is a pet” score (0.8) and low “sound-like” score (0.1).
“meow” has a low “pet-like” score (0.1) and high “sound-like” score (0.9).
“meow” is close to “cat” because “meow” is “cat-related”

Okay, great, I can wrap my head around that. But what if I ask who is considered the most innovative prog rock band?

Word prog score innovative influential
King Crimson 0.9 0.8 0.7
The Mars Volta 0.7 0.6 0.5

It's the same stuff! Just different "dimensions" or columns. So when I hear about how a model has 8 billion parameters would that mean tables with 8 billion columns? No.

When you hear that a model has 8 billion parameters, it doesn’t mean the vector database has 8 billion attributes for each word (or that each word’s vector has 8 billion numbers). Instead, those parameters are part of the LLM itself, not the vector database

Think of an LLM as a super-smart robot chef who knows how to make tons of recipes. The parameters are like the robot’s recipe book; a huge set of instructions that tell it how to mix ingredients (words, ideas, and patterns) to create answers.

For an 8-billion-parameter model, those 8 billion numbers are spread across the LLM’s layers, which are like pages in the recipe book. Each layer helps the robot process words in different ways (understanding grammar, context, or creativity).

Parameters are numbers inside the LLM’s brain that it learned from reading billions of sentences. This is "training."

The LLM’s parameters (those 8 billion numbers) help create the tables above. Part of the LLM’s job is to turn a word like “dog” into a vector by using its recipe book (parameters) to mix patterns it learned from sentences like “Dogs bark.”

The vector database is like a toy box that stores nunbers (vectors for “dog” “cat” “King Crimson” etc.) so the LLM can look them up quickly. The database doesn’t have 8 billion attributes per word. Instead, it stores vectors with a fixed number of dimensions (maybe 300 numbers per word), and there could be millions of vectors (one for each word, phrase, or document).

During training, the LLM looks at billions of sentences and learns patterns, like “dog” appearing with “pet” “bark” and “furry” or “King Crimson” with “prog rock” and “innovative.” It squeezes these patterns into a fixed number of dimensions (300) using math. Each dimension is a blend of features, not a single one. For example, dimension 1 might capture “pet-like + animal-like + a bit of loyalty” while dimension 2 captures “sound-like + noise + a bit of emotion.”

Think of it like a toy box where each toy has a card with 300 stickers, but each sticker’s color is a RGB hex color. You don’t need millions of stickers because the 300 stickers are mixed super smartly to describe everything. Like #AB274F vs #F19CBB they are close, so you can be very nuanced with all the different color choices. In the real system it's not RGB hex but just a floating point number with many decimal places.

So through many many iterations it keeps guessing the right colors and then testing how well it did against the original training data. Eventually some dark greens during more light green as it realized the guess of 0.352423211 wasn't right it's more like 0.352422009
