The early days of scientific Python with Travis Oliphant

Key Points

  • Travis Oliphant created NumPy, which became much more popular than his earlier work because it was more generic and did more
  • The story began in 1999 with a module called NumPy IO, created to load medical imaging data files at Mayo Clinic
  • His first open source project was built by modifying Michael Miller's Table IO library, which had a liberal open source license
  • By 1999, he had created 6-7 different Python packages for scientific computing, collected under "MultiPack"
  • A high school student named Robert Kern built Windows installers for MultiPack, increasing users by a factor of 100
  • Eric Jones from Duke contacted him and they organized the SciPy library in 2001 as a distribution of Python scientific modules
  • SciPy.org became a focal point for the Python scientific ecosystem, along with an annual conference series started in 2001
  • The Hubble Space Science Telescope team adopted Python for image processing using "numeric" arrays
  • Space telescope scientists needed better array capabilities, leading them to create their own "numarray" library
  • A split ecosystem emerged with some libraries using numeric arrays and others using numarray, creating data sharing problems
  • His department chair advised him to spend less time on open source work and more on grants and papers for tenure
  • He started writing NumPy to unite the split communities by combining numeric's features with numarray's capabilities
  • The first 6 months of NumPy development was mostly solo work, stretching his algorithmic and computer science skills
  • It took 6-8 months to write the first alpha release and another year to harden it for public version 1.0
  • NumPy had to support both existing user bases (numeric and numarray), making it a challenging hybrid library
  • Key contributors like Robert Kern and Charles Harris joined after 4 months when they saw he was serious
  • He knew NumPy would be successful when he received a standing ovation from 100 scientific peers at Caltech after his demo
  • Despite NumPy's success, his academic career suffered and he didn't get tenure on his first application, though NumPy became his most successful contribution
